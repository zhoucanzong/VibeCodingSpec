## 高风险变更

涉及以下内容的 spec 应标记为高风险，并要求 review：

- 认证、授权、权限模型。
- 支付、账单、资金相关逻辑。
- PII、隐私数据、敏感日志。
- 外部 API key、token、secret。
- 数据删除、导出、批量更新。
- 安全策略、rate limit、访问控制。

## Review 要求

- [ ] 没有 secret 写入仓库。
- [ ] 输入在边界处校验。
- [ ] 权限检查覆盖关键路径。
- [ ] 错误信息不泄露敏感数据。
- [ ] 日志不记录敏感字段。
- [ ] 依赖或外部服务风险已记录。
