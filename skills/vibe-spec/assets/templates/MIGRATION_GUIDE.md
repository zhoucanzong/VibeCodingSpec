# 迁移指南

用于记录数据库 schema、数据迁移、backfill 和回滚规则。

## 迁移原则

- 优先使用向后兼容迁移。
- 大表变更先评估锁表、耗时和回滚风险。
- 迁移前记录备份或恢复策略。
- Backfill 要可中断、可重试、可观测。
- 删除字段或行为前确认所有消费者已迁移。

## 迁移清单

- [ ] Schema 变更已写入 spec。
- [ ] 迁移顺序明确。
- [ ] 回滚或补救方案明确。
- [ ] 数据校验方式明确。
- [ ] 生产影响窗口明确。
- [ ] 相关测试已记录到 `TESTING_GUIDE.md`。
